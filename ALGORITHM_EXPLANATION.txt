================================================================================
                    LOADOPT 3D LOAD PLANNING SYSTEM
              Algorithm & Rules Explanation (Simple Version)
================================================================================

OVERVIEW
--------
LoadOpt solves the 3D bin packing problem - finding the best way to fit boxes 
into a container (truck/trailer). Think of it like playing Tetris in 3D, but 
with real-world rules about weight, fragility, and stacking.


================================================================================
PART 1: THE PLACEMENT STRATEGY (HOW BOXES ARE POSITIONED)
================================================================================

1. EXTREME POINT ALGORITHM
---------------------------
What it does:
- Instead of randomly trying positions, we use "extreme points" - smart 
  locations where boxes are most likely to fit efficiently.

How it works:
- Start with one point: the back-left-bottom corner of the container (0,0,0)
- When you place a box, create 3 new possible placement points:
  * One ABOVE the box (on top of it)
  * One to the RIGHT of the box (next to it on the X axis)
  * One to the FRONT of the box (next to it on the Y axis)

Simple example:
- You place Box A at the origin (0,0,0)
- System creates 3 new points to try next:
  1. On top of Box A (for stacking)
  2. To the right of Box A (for side-by-side placement)
  3. In front of Box A (for side-by-side placement)

Why this is smart:
- We only try positions where boxes naturally "fit together"
- No wasted space between boxes
- Much faster than trying every possible position

CODE IMPLEMENTATION (from heuristic.py):
```python
# Point above (z+) - Always valid to place on top of a box
if placed.max_z < self.container.height:
    qx, qy, qz = self._quantize_point(placed.x, placed.y, placed.max_z)
    key = (qx, qy, qz)

    qx, qy, qz = self._quantize_point(placed.max_x, placed.y, 0)
    key = (qx, qy, qz)
    if key not in self._points_set:
        new_points.append(PlacementPoint(
            x=qx, y=qy, z=qz,
            available_length=self.container.length - qx,
            available_width=self.container.width - qy,
            available_height=self.container.height - qz
        ))
        self._points_set.add(key)

# Point to the front (y+) - only at ground level
if placed.max_y < self.container.width and placed.z < 0.1:
    qx, qy, qz = self._quantize_point(placed.x, placed.max_y, 0)
    key = (qx, qy, qz)
    if key not in self._points_set:
        new_points.append(PlacementPoint(
            x=qx, y=qy, z=qz,
            available_length=self.container.length - qx,
            available_width=self.container.width - qy,
            available_height=self.container.height - qz
        ))
        self._points_set.add(key)

# Add new points to heap (sorted by height first, then x, then y)
for point in new_points:
    heapq.heappush(self.placement_points, (point.z, point.x, point.y, point))
```


2. ADAPTIVE POINT LIMIT
------------------------
What it does:
- Limits how many placement points we consider to keep the solver fast.

The limits (based on problem size):
- Small problems (≤30 boxes): Try up to 300 points
- Medium problems (31-60 boxes): Try up to 200 points
- Large problems (61-100 boxes): Try up to 150 points
- Very large problems (>100 boxes): Try up to 100 points

Why this matters:
- Trying all points would be too slow
- We try the "best" points first (lowest height, then left-most, then back-most)

CODE IMPLEMENTATION (from heuristic.py):
```python
# Adaptive max points based on problem size (Optimization #3)
n_boxes = len(boxes)
if n_boxes <= 30:
    self._max_points = 300  # More points for thorough search
elif n_boxes <= 60:
    self._max_points = 200
elif n_boxes <= 100:
    self._max_points = 150
else:
    self._max_points = 100  # Still reasonable for large problems

# When placing a box, try limited points first
points_to_try = []
temp_heap = list(self.placement_points)
heapq.heapify(temp_heap)

max_points = min(len(temp_heap), self._max_points)
for _ in range(max_points):
    if not temp_heap:
        break
    _, _, _, point = heapq.heappop(temp_heap)
    points_to_try.append(point)
```


3. PLACEMENT SCORING (CHOOSING THE BEST POSITION)
--------------------------------------------------
When multiple positions are available, we score them:

Score components:
a) Height penalty: Higher positions get worse scores (we prefer ground level)
   - Formula: 10,000 points per meter of height

b) Position preference: Based on delivery order
   - First deliveries go near door (high X), later deliveries at back (low X)
   - Formula: 100 points per meter weighted by delivery order

c) Gap penalty: Leaving empty space is bad
   - If box creates narrow gaps (< 50% of min box dimension), add penalty
   - Formula: 200 points for narrow X gaps, 100 for narrow Y gaps

d) Alignment bonus: Boxes that align with existing boxes get bonus
   - Flush with back/side walls: -100/-50 points (bonus)
   - Aligned with recent box edges: -30/-20 points (bonus)

Best placement = lowest total score

CODE IMPLEMENTATION (from heuristic.py):
```python
def _calculate_placement_score(self, placed: PlacedBox, point: PlacementPoint) -> float:
    """Calculate placement score - lower is better"""
    # Base score: strongly prefer ground level
    z_penalty = point.z * 10000
    
    # Position score: depends on delivery_order
    if placed.box.delivery_order <= 1:
        # First delivery - prefer HIGH X (near door)
        position_score = (self.container.length - point.x) * 100 + point.y
    else:
        # Later deliveries - prefer LOW X (at back)
        position_score = point.x * 100 + point.y
    
    # Future space awareness: penalize placements that create narrow gaps
    gap_penalty = 0.0
    remaining_x = self.container.length - placed.max_x
    remaining_y = self.container.width - placed.max_y
    
    if 0 < remaining_x < self._min_useful_dim * 0.5:
        gap_penalty += 200  # Narrow X gap
    if 0 < remaining_y < self._min_useful_dim * 0.5:
        gap_penalty += 100  # Narrow Y gap
    
    # Alignment bonus
    alignment_bonus = 0.0
    if abs(placed.max_x - self.container.length) < 1.0:
        alignment_bonus -= 100  # Flush with back wall
    if abs(placed.max_y - self.container.width) < 1.0:
        alignment_bonus -= 50   # Flush with side wall
    
    # Check alignment with recent placements
    for existing in self.placements[-5:]:
        if abs(placed.max_x - existing.max_x) < 1.0:
            alignment_bonus -= 30  # Aligned X edges
        if abs(placed.max_y - existing.max_y) < 1.0:
            alignment_bonus -= 20  # Aligned Y edges
    
    return z_penalty + position_score + gap_penalty + alignment_bonus
```


================================================================================
PART 2: VALIDATION RULES (WHAT MAKES A PLACEMENT LEGAL)
================================================================================

1. CONTAINER FIT CHECK
----------------------
Rule: Box must fit completely inside the container
- Box's right edge ≤ container length
- Box's front edge ≤ container width
- Box's top edge ≤ container height
- Box must enter through the door (width ≤ door width, height ≤ door height)


2. COLLISION DETECTION
-----------------------
Rule: Boxes cannot overlap with each other

How it works:
- Uses AABB (Axis-Aligned Bounding Box) method
- Two boxes DON'T collide if ANY of these is true:
  * Box1 is completely to the left of Box2
  * Box1 is completely to the right of Box2
  * Box1 is completely behind Box2
  * Box1 is completely in front of Box2
  * Box1 is completely below Box2
  * Box1 is completely above Box2

Speed optimization:
- Spatial grid divides container into cells
- Only checks boxes in nearby cells (not all boxes)
- O(1) average time instead of O(n)

CODE IMPLEMENTATION (from utils.py):
```python
class CollisionDetector:
    @staticmethod
    def check_collision_fast(box1: PlacedBox, box2: PlacedBox, tolerance: float = 0.01) -> bool:
        """Fast AABB collision check"""
        # Two boxes collide if they overlap in ALL three dimensions
        if box1.max_x <= box2.x + tolerance or box2.max_x <= box1.x + tolerance:
            return False  # Separated on X-axis
        if box1.max_y <= box2.y + tolerance or box2.max_y <= box1.y + tolerance:
            return False  # Separated on Y-axis
        if box1.max_z <= box2.z + tolerance or box2.max_z <= box1.z + tolerance:
            return False  # Separated on Z-axis
        return True  # Overlapping in all three dimensions = collision

class SpatialGrid:
    """3D spatial grid for O(1) collision detection"""
    
    def check_collision(self, placed: PlacedBox) -> bool:
        """Check if a box collides with any existing box - O(1) average case"""
        for existing in self.get_potential_collisions(placed):
            if CollisionDetector.check_collision_fast(placed, existing):
                return True
        return False
    
    def get_potential_collisions(self, placed: PlacedBox) -> List[PlacedBox]:
        """Get boxes in nearby cells only"""
        cells = self._get_cell_range(placed)
        potential_ids = set()
        for cell in cells:
            if cell in self.cells:
                potential_ids.update(self.cells[cell])
        return [self.boxes[bid] for bid in potential_ids if bid in self.boxes]
```


3. SUPPORT VALIDATION (NO FLOATING BOXES)
------------------------------------------
Rule: Every box must have adequate support below it

Requirements:
- Ground-level boxes (z < 0.1cm): Always OK
- Elevated boxes must have:
  * At least 40-50% of their bottom area supported
  * Support boxes must be directly below (within 0.1cm vertically)

Example:
- Box A is 100cm × 50cm (5000 cm² area)
- To place Box B on top, we need:
  * Boxes below that overlap with at least 2000-2500 cm² (40-50%)
  * These boxes must have tops at exactly Box A's bottom height

CODE IMPLEMENTATION (from utils.py):
```python
class StackingValidator:
    @staticmethod
    def check_support_with_grid(box: PlacedBox, grid: SpatialGrid, tolerance: float = 0.1) -> bool:
        """Check support using spatial grid for faster lookups"""
        if box.z < tolerance:
            return True  # On ground
        
        below_boxes = grid.get_boxes_below(box, tolerance)
        if not below_boxes:
            return False
        
        support_area = 0.0
        box_area = box.length * box.width
        
        for placed in below_boxes:
            # Calculate overlapping area
            overlap_x = max(0, min(box.max_x, placed.max_x) - max(box.x, placed.x))
            overlap_y = max(0, min(box.max_y, placed.max_y) - max(box.y, placed.y))
            support_area += overlap_x * overlap_y
            
            if support_area >= 0.4 * box_area:  # 40% minimum
                return True
        
        return support_area >= 0.4 * box_area
```


4. STACKING RULES
------------------
Rule: Heavy and fragile boxes have special stacking restrictions

a) Fragile boxes:
   - Cannot have ANY boxes stacked on top of them
   - Can be placed on top of other boxes

b) Max stack limit:
   - Each box type has a "max_stack" number
   - Example: max_stack=3 means max 3 boxes can be stacked on top

c) Stacking groups:
   - Boxes with different stacking groups cannot stack on each other
   - Example: Food boxes (group "food") cannot be under chemical boxes (group "chem")


5. WEIGHT LIMIT
---------------
Rule: Total weight of all boxes ≤ container's max weight capacity

Additional check:
- Axle weight limits (front axle and rear axle)
- Front axle position: 30% from back
- Rear axle position: 80% from back
- Weight distribution calculated from center of gravity


================================================================================
PART 3: THE OPTIMIZATION PROCESS (FINDING THE BEST SOLUTION)
================================================================================

The system uses a MULTI-PHASE approach with 5 stages:


STAGE 1: FAST HEURISTIC (Initial Quick Solution)
-------------------------------------------------
What it does:
- Tries to place boxes in priority order using the greedy algorithm
- Tests all allowed rotations (6 possible for each box)
- Takes the first valid placement it finds

Sorting priority:
1. Box priority (user-defined importance)
2. Box volume (larger boxes first)

Result: A "good enough" solution in seconds


STAGE 2: GENETIC ALGORITHM (Exploration Phase)
-----------------------------------------------
What it does:
- Treats box order as "DNA" and evolves better orderings

How it works:
1. Create population of 50-100 different box orderings (random)
2. Evaluate each ordering (run heuristic solver)
3. Select best orderings (tournament selection)
4. Breed new orderings:
   - Crossover: Combine two parent orderings
   - Mutation: Randomly shuffle some boxes
5. Repeat for 30-50 generations

Optimizations:
- Caching: Same ordering is never evaluated twice
- Elitism: Top 5-10% always survive to next generation
- Early stopping: Stop if no improvement for 10-15 generations
- Adaptive mutation: Increase mutation if population is too similar

Result: Better box orderings discovered through evolution

CODE IMPLEMENTATION (from optimizer.py):
```python
class GeneticAlgorithmSolver:
    def __init__(self, boxes, container, 
                 population_size=50, generations=30, mutation_rate=0.1,
                 crossover_rate=0.8, elite_size=0.1, early_stopping_patience=10):
        self.boxes = boxes
        self.container = container
        self.population_size = population_size
        self.generations = generations
        self.mutation_rate = mutation_rate
        self.initial_mutation_rate = mutation_rate
        self.crossover_rate = crossover_rate
        self.elite_size = elite_size
        self.early_stopping_patience = early_stopping_patience
        self.cache = FitnessCache()
    
    def solve(self):
        # Create initial population
        population = self.toolbox.population(n=self.population_size)
        
        # Evaluate initial population
        for ind in population:
            ind.fitness.values = self.toolbox.evaluate(ind)
        
        best_fitness = float('-inf')
        generations_without_improvement = 0
        
        for gen in range(self.generations):
            # Select elite
            elite_count = int(self.population_size * self.elite_size)
            elite = tools.selBest(population, elite_count)
            
            # Select parents and breed offspring
            offspring = self.toolbox.select(population, len(population) - elite_count)
            offspring = list(map(self.toolbox.clone, offspring))
            
            # Apply crossover
            for child1, child2 in zip(offspring[::2], offspring[1::2]):
                if random.random() < self.crossover_rate:
                    self.toolbox.mate(child1, child2)
                    del child1.fitness.values
                    del child2.fitness.values
            
            # Apply mutation
            for mutant in offspring:
                if random.random() < self.mutation_rate:
                    self.toolbox.mutate(mutant)
                    del mutant.fitness.values
            
            # Evaluate offspring
            invalid_ind = [ind for ind in offspring if not ind.fitness.valid]
            for ind in invalid_ind:
                ind.fitness.values = self.toolbox.evaluate(ind)
            
            # Replace population (elite + offspring)
            population = elite + offspring
            
            # Track best
            gen_best = max(ind.fitness.values[0] for ind in population)
            if gen_best > best_fitness:
                best_fitness = gen_best
                generations_without_improvement = 0
            else:
                generations_without_improvement += 1
            
            # Early stopping
            if generations_without_improvement >= self.early_stopping_patience:
                break
            
            # Adapt mutation rate based on diversity
            diversity = self._calculate_diversity(population)
            self._adapt_mutation_rate(diversity)
        
        # Return best solution
        best_ind = tools.selBest(population, 1)[0]
        return self._get_full_result(best_ind)
```


STAGE 3: SIMULATED ANNEALING (Refinement Phase)
------------------------------------------------
What it does:
- Fine-tunes the solution by trying small changes

How it works:
1. Start with best solution from genetic algorithm
2. Make small random change (swap 2 boxes in order)
3. If change improves solution: Accept it
4. If change makes it worse: Maybe accept it anyway (based on "temperature")
5. Gradually reduce "temperature" so we accept fewer bad moves
6. Run for 500-1000 iterations

Why accept bad moves?
- Escape local optima (local "good" solutions that aren't the best overall)
- Temperature schedule: Start at 100°, cool down exponentially

Result: Polished solution with local optimizations

CODE IMPLEMENTATION (from optimizer.py):
```python
class SimulatedAnnealingSolver:
    def __init__(self, boxes, container,
                 initial_temp=100.0, cooling_rate=0.95,
                 iterations_per_temp=50, min_temp=0.1):
        self.boxes = boxes
        self.container = container
        self.initial_temp = initial_temp
        self.cooling_rate = cooling_rate
        self.iterations_per_temp = iterations_per_temp
        self.min_temp = min_temp
        self.cache = FitnessCache()
    
    def solve(self):
        n = len(self.boxes)
        
        # Initialize with smart ordering
        current_solution = list(range(n))
        current_solution.sort(key=lambda i: (-self.boxes[i].priority, -self.boxes[i].volume))
        
        current_fitness, current_placements, current_stats = self._evaluate_full(current_solution)
        current_energy = -current_fitness
        
        best_solution = current_solution.copy()
        best_placements = current_placements
        best_stats = current_stats
        best_energy = current_energy
        
        temperature = self.initial_temp
        total_iterations = 0
        max_total_iterations = 500
        
        while temperature > self.min_temp and total_iterations < max_total_iterations:
            for _ in range(self.iterations_per_temp):
                total_iterations += 1
                
                # Generate neighbor (swap, insert, or reverse)
                neighbor = self._get_neighbor_adaptive(current_solution, temperature)
                neighbor_fitness, neighbor_placements, neighbor_stats = self._evaluate_full(neighbor)
                neighbor_energy = -neighbor_fitness
                
                # Metropolis acceptance criterion
                delta_energy = neighbor_energy - current_energy
                
                if delta_energy < 0 or random.random() < math.exp(-delta_energy / max(temperature, 0.01)):
                    current_solution = neighbor
                    current_placements = neighbor_placements
                    current_stats = neighbor_stats
                    current_energy = neighbor_energy
                    
                    if current_energy < best_energy:
                        best_solution = current_solution.copy()
                        best_placements = current_placements
                        best_stats = current_stats
                        best_energy = current_energy
            
            # Geometric cooling
            temperature *= self.cooling_rate
        
        return best_placements, best_stats
```


STAGE 4: EXHAUSTIVE SEARCH (Optional Deep Dive)
------------------------------------------------
What it does:
- Tests many different orderings systematically

Strategies:
a) Priority-based: Try different priority-based orderings
b) Volume-based: Try largest-first, smallest-first
c) Dimension-based: Try longest-first, tallest-first, widest-first
d) Weight-based: Try heaviest-first, lightest-first

Parallel execution:
- Tests multiple orderings at the same time (uses all CPU cores)
- Global result cache prevents duplicate work

Result: Best solution found from diverse ordering strategies


STAGE 5: LOCAL RE-PACKING (Final Polish)
-----------------------------------------
What it does:
- Takes unplaced boxes and tries to fit them in gaps

How it works:
1. Look at boxes that didn't fit
2. Sort by volume (smallest first - easier to fit in gaps)
3. For each unplaced box:
   - Try ALL existing extreme points
   - Try to squeeze it into available space
4. Add any boxes that now fit

Result: Maximum utilization by filling remaining gaps


================================================================================
PART 4: BOX ROTATIONS
================================================================================

Each box can be oriented 6 different ways:

Rotation 0: Original (Length × Width × Height)
Rotation 1: L × H × W (rotated on Y axis)
Rotation 2: W × L × H (rotated on Z axis)
Rotation 3: W × H × L (rotated on Z, then Y)
Rotation 4: H × L × W (rotated on X axis)
Rotation 5: H × W × L (rotated on X, then Z)

Optimization:
- Cubes (all sides equal): Only try 1 rotation (all are the same)
- Symmetric boxes: Skip duplicate rotations
- Respect user's "allowed_rotations" preferences
  (e.g., "This Side Up" boxes have restricted rotations)

CODE IMPLEMENTATION (from utils.py):
```python
class BoxRotation:
    @staticmethod
    def get_dimensions(length: float, width: float, height: float, rotation: int) -> Tuple[float, float, float]:
        """Get dimensions after rotation - with caching"""
        rotations = [
            (length, width, height),   # 0
            (length, height, width),   # 1
            (width, length, height),   # 2
            (width, height, length),   # 3
            (height, length, width),   # 4
            (height, width, length),   # 5
        ]
        return rotations[rotation]
    
    @staticmethod
    def get_unique_rotations(box: Box) -> List[int]:
        """Get unique rotations (avoiding symmetric duplicates)"""
        allowed = [i for i, ok in enumerate(box.allowed_rotations) if ok]
        if not allowed:
            return [0]
        
        # Track unique dimension combinations
        seen = set()
        unique = []
        for rot in allowed:
            dims = BoxRotation.get_dimensions(box.length, box.width, box.height, rot)
            key = tuple(sorted(dims))  # Sort to identify symmetric rotations
            if key not in seen:
                seen.add(key)
                unique.append(rot)
        return unique if unique else [0]
```


================================================================================
PART 5: FITNESS FUNCTION (HOW WE SCORE SOLUTIONS)
================================================================================

A solution is scored by these factors:

1. Utilization (60% of score): How much container volume is used
   - Formula: (Total box volume / Container volume) × 100
   - Goal: Maximize

2. Placement count (40% of score): How many boxes were placed
   - Formula: (Placed boxes / Total boxes) × 100
   - Goal: 100%

3. Balance penalty: Weight distribution
   - Check if center of gravity is near container center
   - Penalty (0.2) if weight is too far forward/backward/sideways
   - Goal: Balanced load

4. Validation penalty: Rule violations
   - Penalty (0.1 per error) for validation errors
   - Penalty (0.5) for axle overload

Combined formula:
Fitness = (0.6 × utilization) + (0.4 × placement_rate) - penalties

CODE IMPLEMENTATION (from optimizer.py):
```python
def _calculate_fitness(self, stats: dict) -> float:
    """Calculate fitness from stats"""
    utilization = stats['utilization_pct'] / 100.0
    placed_ratio = stats.get('placed_count', 0) / max(1, len(self.boxes))
    
    # Penalties
    penalty = 0.0
    
    if stats.get('weight_distribution'):
        wd = stats['weight_distribution']
        cog = wd['center_of_gravity']
        
        # Balance penalty
        center_y = self.container.width / 2
        balance_deviation = abs(cog['y'] - center_y) / self.container.width
        penalty += balance_deviation * 0.2
        
        # Axle overload penalties
        if self.container.front_axle_limit and wd['front_axle'] > self.container.front_axle_limit:
            penalty += 0.5
        if self.container.rear_axle_limit and wd['rear_axle'] > self.container.rear_axle_limit:
            penalty += 0.5
    
    if not stats.get('is_valid', True):
        penalty += len(stats.get('validation_errors', [])) * 0.1
    
    return max(0.0, utilization * 0.6 + placed_ratio * 0.4 - penalty)
```


================================================================================
PART 6: PERFORMANCE OPTIMIZATIONS
================================================================================

1. SPATIAL GRID
---------------
- Divides container into 3D grid of cells
- Cell size adapts to box sizes (1.8× median box dimension)
- Collision checks only look at nearby cells
- Speed: O(1) instead of O(n)

CODE IMPLEMENTATION (from utils.py):
```python
class SpatialGrid:
    """3D spatial grid for fast collision detection - O(1) lookups instead of O(n)"""
    
    def __init__(self, length: float, width: float, height: float, cell_size: float = 50.0):
        self.cell_size = cell_size
        self.grid_x = int(np.ceil(length / cell_size)) + 1
        self.grid_y = int(np.ceil(width / cell_size)) + 1
        self.grid_z = int(np.ceil(height / cell_size)) + 1
        self.cells: Dict[Tuple[int, int, int], Set[int]] = {}
        self.boxes: Dict[int, PlacedBox] = {}
        self._box_cells: Dict[int, Set[Tuple[int, int, int]]] = {}
    
    def _get_cell_range(self, placed: PlacedBox) -> List[Tuple[int, int, int]]:
        """Get all cells that a box occupies"""
        cells = []
        min_x = int(placed.x / self.cell_size)
        max_x = int(placed.max_x / self.cell_size)
        min_y = int(placed.y / self.cell_size)
        max_y = int(placed.max_y / self.cell_size)
        min_z = int(placed.z / self.cell_size)
        max_z = int(placed.max_z / self.cell_size)
        
        for x in range(min_x, max_x + 1):
            for y in range(min_y, max_y + 1):
                for z in range(min_z, max_z + 1):
                    cells.append((x, y, z))
        return cells
    
    def add_box(self, placed: PlacedBox) -> None:
        """Add a box to the spatial grid"""
        box_id = id(placed)
        self.boxes[box_id] = placed
        cells = self._get_cell_range(placed)
        self._box_cells[box_id] = set(cells)
        
        for cell in cells:
            if cell not in self.cells:
                self.cells[cell] = set()
            self.cells[cell].add(box_id)
    
    def get_potential_collisions(self, placed: PlacedBox) -> List[PlacedBox]:
        """Get boxes that could potentially collide - only checks nearby cells"""
        cells = self._get_cell_range(placed)
        potential_ids = set()
        
        for cell in cells:
            if cell in self.cells:
                potential_ids.update(self.cells[cell])
        
        return [self.boxes[bid] for bid in potential_ids if bid in self.boxes]
```


2. GLOBAL ORDER CACHE
----------------------
- Remembers results of box orderings
- If we try the same ordering twice, reuse cached result
- Saves massive computation time in genetic algorithm
- Thread-safe with locks for parallel processing

CODE IMPLEMENTATION (from optimal_solver.py):
```python
class GlobalOrderCache:
    """Global LRU cache for order evaluations - shared across all phases."""

    def __init__(self, max_size: int = 50000):
        self.cache: OrderedDict[str, Tuple[float, List[PlacedBox], dict]] = OrderedDict()
        self.max_size = max_size
        self.hits = 0
        self.misses = 0

    def _get_key(self, order: List[int]) -> str:
        """Generate cache key from order (use hash for speed)"""
        return str(hash(tuple(order)))

    def get(self, order: List[int]) -> Optional[Tuple[float, List[PlacedBox], dict]]:
        """Get cached result if available (LRU - moves to end)"""
        key = self._get_key(order)
        if key in self.cache:
            self.hits += 1
            self.cache.move_to_end(key)  # LRU behavior
            return self.cache[key]
        self.misses += 1
        return None

    def set(self, order: List[int], fitness: float, placements: List[PlacedBox], stats: dict):
        """Store result in cache with LRU eviction"""
        key = self._get_key(order)
        if key in self.cache:
            self.cache.move_to_end(key)
        else:
            if len(self.cache) >= self.max_size:
                self.cache.popitem(last=False)  # Remove oldest
        self.cache[key] = (fitness, placements, stats)

    @property
    def hit_rate(self) -> float:
        """Calculate cache hit rate"""
        total = self.hits + self.misses
        return self.hits / total if total > 0 else 0.0
```

PART 7: COMMON SCENARIOS & SOLUTIONS
================================================================================

Scenario 1: "Boxes are floating in the air"
Solution: Support validation requires 40-50% overlap within 0.1cm tolerance
- Lowered from initial 80% to 40-50% to allow more placements
- Extreme points only created at ground (z=0) or on existing box tops
- Lateral points (X+, Y+) only created at ground level for proper support

Scenario 2: "Only 40% of boxes placed"
Causes:
- Door size too small (boxes can't enter)
- Weight limit reached
- No valid placement points remain
- Stacking rules too restrictive
Solutions:
- Adaptive max placement points to try (100-300 based on problem size)
- Added fallback to try ALL points if initial attempts fail
- Relaxed support requirement to 40-50%
- Lazy deletion with fragmentation tracking for point management

Scenario 3: "Boxes overlap/collide"
Solution: Spatial grid collision detection + final validation
- Every placement is checked before accepting
- Final result is validated and collisions are removed
- Thread locks prevent race conditions in parallel processing

Scenario 4: "Takes too long to solve"
Solutions:
- Adaptive point limits (fewer points for larger problems)
- Caching (never compute same ordering twice)
- Early stopping (stop if no improvement)
- 30-minute timeout on Celery tasks
