# 5. **Algorithms Specification**

## **Heuristic Algorithm (FAST Preview Mode)**

1. Sort SKUs by volume descending.
2. Generate placement points (corners).
3. For each SKU instance:

   * Try all allowed rotations.
   * Check fit in container.
   * Check stacking rules + fragility.
   * Place in best-fit corner.
4. Update placement points.

Complexity must allow near-instant response (<1s for 200 items).

---

## **Optimization Algorithm (IMPROVED Mode)**

### **Genetic Algorithm**

Population structure:

```
Individual = ordering of SKU instances
Fitness = combination of:
  - utilization efficiency
  - weight balance penalty
  - stacking violations penalty
  - door-entry violations penalty
```

Operators:

* Crossover: ordered crossover
* Mutation: swap indices
* Elitism: preserve top 5%

Termination:

* N generations or no improvement.

---

## **Simulated Annealing**

Neighborhood moves:

* Swap two SKUs
* Rotate an SKU
* Move SKU to different corner

Cooling schedule: exponential.

---

## **Validation Layer**

Must check:

* Item inside container
* Item fits through door
* No collisions
* Max stacking limits
* Fragile items only on top
* No SKU placed on incompatible SKU
* Weight distribution acceptable
* Axle loads within limits

If violations exist → mark plan invalid.
